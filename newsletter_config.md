## What Has Been Built (Current State)

### The Tool
- `tools/newsletter.py` — one command does everything:
  1. Asks for a topic
  2. Calls Groq AI (free, instant) to research and write all content
  3. Auto-generates an eye-catching subject line
  4. Builds a Gmail-compatible HTML newsletter
  5. Creates a Gmail draft via Gmail API
  6. Opens the draft in Chrome — user adds recipients and hits Send

### How to Run
- Open `tools/newsletter.py` in VS Code
- Click the ▶ play button
- Type the topic when asked
- Gmail opens in Chrome with newsletter pre-filled

### AI Stack (all free)
- Content generation: **Groq API** (Llama 3.3 70B) — instant, 14,400 req/day free
- Gmail integration: **Gmail API** (Google OAuth) — free
- No paid APIs. No rate limits in normal use.

## CONTRAST — ABSOLUTE HARD RULE (never violate)
- Text and background must always achieve a minimum 4.5:1 WCAG contrast ratio (target 7:1)
- Text must NEVER be the same hue/lightness as the surface behind it
- Dark brand colors (low lightness) may ONLY be used for backgrounds/surfaces — never for text, numbers, or accents
- Light/vivid brand colors must fill the ACCENT role: stat numbers, section labels, borders, highlights
- If a chosen text color fails contrast against its background, automatically replace it with the nearest readable color
- Readability ALWAYS overrides brand matching — a readable off-brand color beats an invisible on-brand one
- This rule is enforced in code via WCAG contrast checking on every text/background pair

## Color Palette Rule
DYNAMIC — topic-driven. LIGHT PREMIUM BASE.
- Page background: warm off-white cream (#FAF7F0). Cards: pure white (#FFFFFF).
- Only the header masthead band stays dark (brand dark color) — everything else is light.
- Body text = near-black dark (#1A202C family). Readable in both light and dark mode.
- Accent color = the topic's MOST VIVID/saturated brand color, always. Used for: stat numbers, section labels, left-border bars, stat top-borders, bar fills, table header borders, CTA button.
- Dark brand colors = header background ONLY — never used for text, numbers, or accents.
- A small known-brands dictionary provides verified palettes for well-known topics (FIFA, Spotify, Apple, etc.) — used first, before web extraction.
- Contrast guard: if a vivid accent is too light for white, it is darkened within its own hue until it passes 4.5:1 — never replaced with a different hue.
- Light base survives dark-mode inversion far better than a fully dark design.
- Always send one real test to Gmail mobile with dark mode ON to confirm readability.
- Meta tags included: color-scheme light dark, supported-color-schemes light dark.

## Accuracy Rules (non-negotiable)
- Never fabricate events that have not happened yet
- Today's date must be respected — do not invent future outcomes
- Future events: cover only confirmed facts, schedule, history, qualified teams
- Projections must be clearly labelled as "Projected" or "Expected"
- Every statistic must be real and verifiable
- If unsure whether something happened — leave it out

## Visual Design System (Batch 1 — applied)

### Type Scale
- Title: 44px Georgia serif, bold
- Subtitle: 20px Georgia serif, regular
- Stat numbers: 34px Arial, 800 weight
- Stat labels: 13px Arial, bold
- Body text: 15px Arial, line-height 1.7–1.85
- Section eyebrows: 11px Arial, bold, letter-spacing 3px, small-caps uppercase

### Header Metadata Row
- Sits under subtitle in header: `ISSUE 001 · JUNE 2026 · 5 MIN READ · INTELLIGENCE BRIEF`
- 11px Arial, letter-spacing 2px, uppercase, muted color

### Section Labels
- All section eyebrow labels use a 3px colored left-tick (border-left)
- Consistent small-caps style across all sections

### Spacing
- Section vertical padding: 46px top (not 36px)
- Stat note text: uses muted text color (#9aa3b0 minimum) — never below 4.5:1 contrast

### AI Stack Label
- Content is generated by Groq AI (Llama 3.3-70b) — not Claude AI
- All comments and print statements reflect this correctly

## Newsletter Structure (every newsletter must have these sections)
1. **Header** — title, subtitle, sources, date
2. **Executive Summary** — 3 paragraphs of essential context
3. **By The Numbers** — 9 stat cards in a 3-column grid
4. **Timeline** — 10 chronological events, alternating row colors
5. **Data Table** — topic-specific comparison or breakdown table
6. **Outlook & Analysis** — 3 forward-looking sections (Short Term / Key Risk / Watch For)
7. **Footer** — sources credited, WAT Framework attribution

---

## Subject Line Rules
- Under 55 characters
- Must create urgency or curiosity
- Must reflect actual content — no clickbait
- Never state unconfirmed future events as fact

---

## Tone
- Professional and authoritative
- Specific numbers and facts — never vague statements
- Short punchy headlines
- Detailed but scannable body text

---

## Gmail Compatibility (never break these)
- All styles inline only — no `<style>` blocks (Gmail strips them)
- Tables only for layout — no flexbox or grid
- No JavaScript
- No external fonts — Arial, Georgia, Calibri only
- Max width 680px

## Known Limitation — Data Freshness
- Groq/Llama has a training cutoff and no live data access.
- For time-sensitive topics (earnings, prices, scores, live events), figures may be dated.
- The prompt instructs the model to label stale data with its year and omit fabricated financials.
- Future upgrade path: add a web-search/data-fetch step before generation to ground stats in current data.
